# A9 细粒度情感分析与舆情监测平台 - 代码框架与 Prompt 交互文档

## 一、 目录结构与系统框架概述

本模块（A9）旨在提供一个基于预训练小型语言模型（Hugging Face Pipeline）的情感分析与舆情监控 Web 平台。

**当前 A9 目录结构：**
```text
A9/
├── app.py               # Streamlit 主程序，包含交互界面、模型推理与 Plotly 可视化逻辑
├── requirements.txt     # 核心依赖环境锁定（Streamlit, transformers, plotly, torch等）
└── prompts.md           # 本文档，即代码框架解读与模型交互指南
```

## 二、 详细模块与函数解读

整个 `app.py` 按照从上到下的执行流和功能区划分为三个主要模块（Tabs）以及一个全局模型加载函数。

### 1. 全局配置与模型加载器
*   **全局 CSS 样式**：通过 `st.markdown(..., unsafe_allow_html=True)` 注入自定义的 HTML/CSS，实现主标题高亮、说明卡片的定制化UI（如 `expl_card` 样式）。
*   **`load_sentiment_model()` 函数**：
    *   **功能**：初始化 Hugging Face 情感分析管道 (`pipeline`)。
    *   **核心逻辑**：使用 `st.cache_resource` 装饰器，确保模型在 Streamlit 重新运行应用时不会被反复加载。该函数挂载了由 lxyuan 提供的 `distilbert-base-multilingual-cased-sentiments-student` 蒸馏模型，此模型支持多种语言并能对文本做三分类（Positive, Negative, Neutral）。

### 2. 交互模块分镜 (Tabs)
*   **模块 1：基础分类与置信度 (`tab1`)**
    *   **解读**：收集用户的自然语言输入并传送给分类器，返回最可能的标签(label)以及置信度(score)。
    *   **可视化**：使用 `plotly.graph_objects.Indicator` 绘制仪表盘 (Gauge Chart)。根据极性的不同，仪表盘进度条会自动映射相应的颜色（如正面为绿色，负面为红色）。此模块强调了“置信度”在 AI 判断中的参考价值。

*   **模块 2：显式情感 vs 隐式情感识别 (`tab2`)**
    *   **解读**：提供双栏对照实验区。重点阐释自然语言处理（NLP）的一大难点——隐式情绪模型捕捉。
    *   **逻辑**：一栏展示具有明显褒贬词的“显式文本”，另一栏展示仅陈述事实却暗含负面抱怨的“隐式文本”。通过直接调用同一个 `classifier` 对两者进行对比推理。

*   **模块 3：舆情挖掘与可视化仪表盘 (`tab3`)**
    *   **解读**：模拟企业级宏观舆情的数据操作，演示了从单句推断向批量数据集（Batch）处理的跃迁。
    *   **逻辑**：系统内部硬编码了 15 条模拟电商评论数据（覆盖品控、物流、系统故障等维度）。执行批量推理后，数据被注入 `pandas.DataFrame` 进行结构化统计。使用 `plotly.graph_objects.Pie` 绘制一个甜甜圈比例图，直观展现品牌整体口碑极性分布。

## 三、 典型 Prompt 交互示例与测试指南

在这里，“Prompt”（提示词或输入文本）直接代表真实的商品评论。你可以使用以下示例输入到 **模块1** 和 **模块2** 中，观察模型的反应与置信度变化：

### 1. 显式情感 (Explicit Sentiment)
*   **输入 Prompt (Positive)**："这款手机的屏幕色彩非常艳丽，拍照也很清晰，非常满意！"
    *   **预期输出**：Positive (置信度 > 95%)
*   **输入 Prompt (Negative)**："什么垃圾产品，买回来第一天按键就失灵了，客服态度还极差！"
    *   **预期输出**：Negative (置信度 > 90%)
*   **输入 Prompt (Neutral)**："今天上午收到了快递，包装里有一台手机和一条充电线。"
    *   **预期输出**：Neutral（陈述事实，无情绪爆发）。

### 2. 隐式情感 (Implicit Sentiment / 沙盒测试重点)
隐式情感考验的是模型对情境和客观缺陷的理解。
*   **输入 Prompt (隐式缺点 1)**："玩游戏不到半小时，后盖就可以用来煎鸡蛋了。"
    *   **预期测试点**：模型是否能理解“煎鸡蛋”在此处意味着“发热严重”从而给出 Negative，还是会被困在字面意思给出 Neutral。
*   **输入 Prompt (隐式缺点 2)**："在太阳底下根本看不清屏幕上的字。"
    *   **预期测试点**：没有任何贬义词，但抱怨了屏幕亮度不足。
*   **输入 Prompt (隐式优点 1)**："给我省去了带两部手机出门的麻烦。"
    *   **预期测试点**：暗示了双卡双待或者功能全面，看模型是否能判定为 Positive。

### 3. 多情绪混合 (Mixed Sentiment / 边界测试)
*   **输入 Prompt**："物流送货确实很快，手机外观也很漂亮，就是续航太让人崩溃了，半天就要充一次电。"
    *   **预期发现**：这段话前两句是正面，最后一句是强烈的负面。你可以观察这个模型最终判断出的是整体偏负面还是偏正面？同时观察它的**置信度**是否因为情感的矛盾而显著降低（例如降到 50%~60%），这就是该项目展示“置信度量化”的价值所在。
